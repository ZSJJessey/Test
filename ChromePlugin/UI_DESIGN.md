# 网页摘录插件 · UI 设计说明

## 1. 文档信息

| 项目 | 说明 |
|------|------|
| 文档名称 | Chrome 插件 UI 设计说明 |
| 关联需求 | `REQUIREMENTS.md` v0.1 |
| 版本 | v0.1 |
| 约束 | 所有实现文件与代码统一放在 **`ChromePlugin/`** 目录下 |

## 2. 设计目标与原则

- **快**：摘录保存与复制路径步骤少；列表一眼可见正文预览。
- **稳**：操作结果有明确反馈（成功 / 失败 / 需确认）；危险操作二次确认。
- **轻**：Popup 为主界面；遵循 Manifest V3；样式克制，适配窄屏（见 §3）。
- **可达**：关键按钮有可见文案或 `aria-label`；Tab 顺序合理；复制可通过键盘触发（对齐需求 NF-03）。

## 3. 载体与尺寸

| 载体 | 用途 | 建议 |
|------|------|------|
| **浏览器工具栏 Popup** | 列表浏览、筛选、搜索、复制、保存选中、分类管理入口 | **主界面**；建议最小宽度 **360px**，高度随内容滚动，避免固定超高导致溢出困难 |
| **右键菜单** | 选中后「保存到摘录」 | 文案简短，与 Popup 内保存流程一致 |
| **页面内浮动按钮（可选）** | 快速保存选中 | 小圆角按钮 + 图标；不遮挡正文；可关闭（后续可在选项里配置） |
| **保存确认层** | 选分类、校验长度、展示来源预览 | 可在 Popup 内以**顶部折叠区 / 模态层 / 独立步骤**实现，保持同一文档内组件复用 |

**说明**：若后续采用侧边栏（Side Panel），布局规则与 Popup 同一套组件即可横向加宽，本期以 Popup 为准。

## 4. 信息架构

```
Popup
├── 顶部栏：标题「摘录」+ 可选「保存当前选中」+ 可选设置/更多
├── 工具区：分类筛选 | 搜索框 | 排序（可选，P2）
├── 列表区：摘录卡片列表
└── 底部/浮动：新建分类、管理分类（或收入「更多」菜单）

保存流程（从选中触发）
└── 正文预览（截断）+ 分类选择（必选）+ 新建分类 + 确认保存

分类管理（子视图或二级页）
└── 列表：分类名 + 条目数 | 重命名 | 删除（确认策略见需求 F-12）
```

## 5. 界面分区（Popup 主界面）

### 5.1 顶部栏

| 元素 | 说明 |
|------|------|
| 标题 | 固定文案如「摘录」，可加条目总数徽章（可选） |
| 主操作 | 「保存当前选中」：无选中或不可读时 **禁用**，悬停提示原因 |
| 次要入口 | 「分类管理」或「⋯」收纳：排序方式、清空数据（P2）、关于 |

### 5.2 筛选与搜索

| 元素 | 说明 |
|------|------|
| 分类筛选 | 下拉或分段控件：**全部** + 各分类 + **未分类/默认**（对齐 F-13） |
| 搜索 | 单行输入，占位符如「搜索正文或标题」；清空按钮；无结果时展示空状态文案 |

### 5.3 摘录列表（卡片）

每条卡片建议包含：

| 区块 | 内容 | 备注 |
|------|------|------|
| 主文 | 摘录正文，**2～4 行预览**，过长省略号 | 单条上限需求见 F-02；超长保存时的裁剪提示在保存流程内说明 |
| 元信息（P1） | 一行小字：保存时间、来源标题（截断）、可点链接图标打开来源 URL | 无标题时显示域名或「未知标题」 |
| 分类标签 | 展示当前分类名；可快捷改分类（P1 若做：下拉或侧滑，否则收入详情） |
| 操作 | **复制**（主按钮）、**删除**（次要，需确认） |

**状态**：

- **加载**：列表骨架或简短「加载中」。
- **空列表**：区分「无任何摘录」与「当前筛选/搜索无结果」两套文案与图示。
- **复制成功**：该行按钮短暂变为「已复制」或 Toast；失败时错误提示（权限、剪贴板不可用）。

### 5.4 保存摘录流程 UI

触发：右键菜单 / Popup「保存当前选中」/ 快捷键 /（可选）浮动按钮。

| 步骤 | 界面要点 |
|------|----------|
| 校验 | 无选中：Toast 或内联提示「请先在页面选中文字」 |
| 长度 | 接近或超过上限（如 2000 字）：提示**裁剪保存**或**取消**，并展示将保存的字数 |
| 预览 | 展示将要保存的正文预览（可滚动） |
| 分类 | **必选**：下拉选已有分类；「+ 新建分类」展开输入框（名称校验：非空、长度、非法字符，对齐 F-11） |
| 确认 | 主按钮「保存」；取消关闭流程 |

**重复保存（P2）**：若实现去重或询问，使用对话框：「已存在相似摘录，是否仍保存？」

## 6. 分类管理界面

入口：Popup 内「分类管理」。

| 功能 | UI 行为 |
|------|---------|
| 新建 | 行内输入 + 确认；错误时在输入旁展示校验信息 |
| 重命名 | 行内编辑或弹层；失焦/确认保存 |
| 删除 | 危险操作：**二次确认**；文案说明条目处理方式（迁入默认 / 一并删除 / 非空禁止，与产品最终约定一致） |
| 默认/未分类 | 始终可见；删除策略不与「普通分类」冲突 |

## 7. 右键菜单（系统菜单）

- 菜单项文案：**保存到摘录**（或与品牌一致的短名）。
- 点击后与 Popup 内保存流程一致（若扩展未打开，可先打开 Popup 再进入保存步骤，或注入轻量页面——以实现为准，UI 逻辑保持一致）。

## 8. 可选：页面浮动按钮

- 位置：默认右下角；小尺寸；半透明背景以免挡阅读。
- 点击：等同触发保存流程；未选中时提示与 Popup 一致。
- **关闭**：长按或设置中关闭（本期可只做关闭开关占位）。

## 9. 视觉与文案规范（简要）

| 类别 | 建议 |
|------|------|
| 字体 | 使用系统字体栈，正文不小于 **13px**，辅助信息 **12px** |
| 色彩 | 中性背景 + 单一主色用于主按钮与链接；错误红仅用于错误与删除确认 |
| 间距 | 8px 基准网格；卡片间距统一 |
| 图标 | 复制、链接外链、删除、搜索等沿用 Chrome 常见隐喻，附 `aria-label` |
| 文案 | 按钮用动词：**复制**、**保存**、**删除**；禁用状态配合说明 |

## 10. 无障碍与键盘（NF-03）

- 列表项内「复制」可通过 **Tab** 聚焦，**Enter** 触发。
- 搜索框、分类下拉、保存表单控件顺序符合自上而下阅读顺序。
- 模态对话框焦点陷阱与 Esc 关闭（若采用模态）。

## 11. 与里程碑的 UI 对应

| 里程碑 | UI 范围 |
|--------|---------|
| M1 | Popup：默认分类、列表、复制反馈、保存流程（单分类可选简化） |
| M2 | 多分类筛选、搜索条、来源标题/URL 展示 |
| M3 | 导出/导入、去重对话框、体验细化（排序、空状态优化） |

本插件后续 **manifest、脚本、样式、图标、构建产物** 等均放在 **`ChromePlugin/`** 下，与本设计文档及 `REQUIREMENTS.md` 保持同目录，便于版本管理与审查。
