# Chrome 浏览器插件：网页摘录（选中保存 + 分类 + 复制使用）

## 1. 文档信息

| 项目 | 说明 |
|------|------|
| 文档名称 | 网页摘录插件需求说明 |
| 版本 | v0.1 |
| 状态 | 草案 |

## 2. 背景与目标

用户在浏览网页时，经常需要**临时记下某一句话**，事后在写作、回复、整理笔记时**快速复制粘贴**使用。本插件用于在浏览器内完成 **选中 → 保存 → 分类 → 检索 → 一键复制**，避免依赖外部笔记工具或反复查找历史页面。

### 2.1 目标

- 支持用鼠标**选中页面中的一段文本**（以「一句话」为主的短文本）并保存。
- 保存时可选择或新建**分类**，便于后续查找。
- 提供统一入口，在需要时**复制到剪贴板**，可直接粘贴使用。

### 2.2 非目标（本期可不实现）

- 跨设备云端同步（除非单独立项）。
- 全文网页剪辑、富文本完整排版还原。
- OCR 识别图片中的文字。

## 3. 用户与场景

### 3.1 典型用户

- 需要收集金句、论点、邮件/文档模板句式的办公与学习用户。

### 3.2 核心场景

1. **摘录**：阅读文章 → 选中一句话 → 触发保存 → 选分类 → 确认保存。
2. **复用**：写文档或回复时 → 打开插件面板 → 找到条目 → 一键复制 → 粘贴。
3. **整理**：为历史条目调整分类、删除无用条目。

## 4. 功能需求

### 4.1 文本选中与保存

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F-01 | 用户在网页上用鼠标选中一段文本后，可通过插件提供的入口触发「保存摘录」（例如右键菜单、浮动按钮、快捷键，具体交互见第 6 节）。 | P0 |
| F-02 | 保存内容应为**纯文本**（去除多余首尾空白可选配置）；长度建议设置上限（如单条 ≤ 2000 字），超出时提示裁剪或拒绝。 | P0 |
| F-03 | 保存时记录**可选元数据**：保存时间、来源页面标题、来源 URL（便于回溯）。 | P1 |
| F-04 | 同一段文本重复保存时，可提示「已存在」或允许保存为多条（产品二选一，建议默认去重或询问）。 | P2 |

### 4.2 分类

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F-10 | 保存时必须选择**一个分类**；支持从已有分类中选择。 | P0 |
| F-11 | 支持**新建分类**（名称校验：非空、长度上限、非法字符限制）。 | P0 |
| F-12 | 支持分类的**重命名、删除**（删除分类时需约定：条目迁入默认分类 / 一并删除 / 禁止删除非空分类）。 | P1 |
| F-13 | 支持「未分类」或默认分类，避免用户被强制先建分类才能保存。 | P1 |

### 4.3 浏览、检索与复制

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F-20 | 插件弹窗（Popup）或侧边栏中展示摘录列表，可按**分类筛选**。 | P0 |
| F-21 | 支持**关键词搜索**（标题/正文）。 | P1 |
| F-22 | 每条摘录提供**复制到剪贴板**按钮；复制成功后给出明确反馈（Toast / 文案变更）。 | P0 |
| F-23 | （可选）支持最近使用排序 / 按时间排序。 | P2 |

### 4.4 数据与隐私

| ID | 需求描述 | 优先级 |
|----|----------|--------|
| F-30 | 默认数据存储在浏览器本地（如 `chrome.storage.local`），不上传服务器。 | P0 |
| F-31 | 提供**清空全部数据**或导出/导入（JSON）作为后续迭代项。 | P2 |

## 5. 非功能需求

| ID | 类别 | 说明 |
|----|------|------|
| NF-01 | 性能 | 列表加载与搜索在常规数据量（如 < 5000 条）下流畅可用。 |
| NF-02 | 兼容性 | 支持当前稳定版 Chrome（Manifest V3）。 |
| NF-03 | 无障碍 | 弹窗内按钮具备可读标签；主要操作可用键盘完成（至少复制）。 |
| NF-04 | 安全 | 仅在必要权限下访问页面选中内容；不向第三方发送摘录内容。 |

## 6. 交互与权限（建议）

### 6.1 触发保存的方式（任选组合）

- **右键菜单**：选中文字 → 右键 →「保存到摘录」→ 弹出分类选择后保存。
- **浏览器工具栏 Popup**：选中后打开 Popup，点击「保存当前选中」（需 `activeTab` + `scripting` 读取选中内容，或依赖用户粘贴）。
- **快捷键**：可配置全局/页面快捷键触发保存流程。

说明：从页面读取选中文本通常需要 `activeTab` 或在特定站点上的 `scripting`/`host_permissions`；具体以最终实现为准，在 manifest 中写明。

### 6.2 权限清单（草案）

- `storage`：本地保存摘录与分类。
- `contextMenus`：若采用右键菜单。
- `activeTab`：按需读取当前标签页选中文本（配合脚本注入策略）。
- `clipboardWrite`：若使用 Clipboard API 复制（部分场景也可用 `document.execCommand('copy')` 替代）。

## 7. 验收标准（示例）

1. 任意网页选中一句话，完成保存并选择分类后，在 Popup 列表中可见，且正文与分类正确。
2. 点击复制后，剪贴板内容与列表一致，粘贴到记事本无误。
3. 刷新浏览器 / 重启浏览器后数据仍存在（本地持久化）。
4. 分类新建、重命名、删除行为符合第 4.2 节约定且无数据静默丢失。
