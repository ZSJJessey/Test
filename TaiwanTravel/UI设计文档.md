# 台湾 6 天 5 晚自由行互动攻略 — UI 设计文档

> 依据 `REQUIREMENTS.md`（PRD、交互、可视化、图片与验收）与 `页面清单模块.md`（模块拆分与文案要点）整理，供静态页（HTML/CSS/JS）实现与视觉走查使用。

---

## 1. 文档作用与范围

| 项 | 说明 |
| --- | --- |
| 产品形态 | 静态单页，优先移动端，兼容桌面端 |
| 技术约束 | 原生 HTML/CSS/JS，资源相对路径，入口 `index.html`，可部署 GitHub Pages |
| 本文覆盖 | 视觉语言、布局、各模块 UI 结构、状态与动效、图片层级；路线区含 Google iframe（非 JS API）；总览含预算计算器 |
| 文案对齐 | 对外主文案以 `页面清单模块.md` 为准；`REQUIREMENTS.md` 中补充字段（如标签「自由行 / 路线清晰…」）作为可选增强，实现时二选一或合并展示需产品确认 |

---

## 2. 设计原则与体验目标

- **信息清晰优先于装饰**：模块边界清楚，扫描路径短（5 秒内理解页面性质，15 秒内可定位路线与准备事项）。
- **风格关键词**：清爽、旅行感、杂志感、轻量信息设计；避免廉价旅游模板风（少用高饱和渐变大块、少通用「飞机护照」剪贴画风）。
- **互动服务理解**：图片、时间线、路线节点与文案联动，而非纯展示堆砌。

## 3. 设计令牌（Design Tokens）

### 3.1 色彩

| 角色 | 建议值（可 CSS 变量） | 用途 |
| --- | --- | --- |
| 页面底 | 米白系 `#F7F5F0` ~ `#FAF8F3` | 主背景，减轻长文疲劳 |
| 表面/卡片 | `#FFFFFF` 或略暖白，透明度用于 Hero 信息层 | 卡片、浮层 |
| 主色（海岛蓝） | `#2B6CB0` ~ `#3182CE` 区间选一主色 | 主按钮、链接、当前节点、选中态 |
| 辅色（山景绿） | `#2F855A` ~ `#38A169` | 次级强调、部分标签、正向提示 |
| 正文 | `#2D3748` ~ `#1A202C` | 正文主色 |
| 次要文字 | `#718096` | 辅助说明、元信息 |
| 分割线/描边 | `#E2E8F0` | 卡片边、时间线 |
| 必做（语义） | 偏红或琥珀强调 `#C05621` / `#DD6B20` | 「必做」类标签（与 PRD「红黄绿」一致时：必做偏警示暖色） |
| 可选 | 中性蓝灰或主色浅色阶 | 「可选」 |
| 避坑 | 绿色系或深绿（与「安全通过」区分时可用深青 `#285E61`） | 「避坑」——实现时保持三色可区分即可，避免红绿色盲不可辨 |

> PRD 要求红黄绿区分「必做 / 可选 / 避坑」：实现时用**色相 + 图标/文案**双重编码，不单靠颜色。

### 3.2 字体

| 层级 | 建议 |
| --- | --- |
| 中文标题 | 系统栈：`"PingFang SC", "Microsoft YaHei", sans-serif`；字重 600–700；Hero 主标题可略加大字距（tracking）营造海报感 |
| 英文/数字 | 可与标题同族，或使用 `system-ui` 统一 |
| 正文 | 字重 400–500，行高 1.5–1.65；段落宽度在桌面端限制 `max-width`（如 36–42em）提升可读性 |
| Day 标签 | 字重 600，与正文对比明显 |

### 3.3 间距与圆角

- **基准间距**：4px 网格；模块纵向间距移动端 `24–32px`，桌面 `40–56px`。
- **卡片内边距**：移动端 `16–20px`，桌面 `20–24px`。
- **圆角**：卡片 `12–16px`；按钮全圆角（pill）或 `10–12px`；路线节点圆形。

### 3.4 阴影与层次

- 卡片默认：`0 1px 3px rgba(0,0,0,.06)`；hover 略升：`0 4px 12px rgba(0,0,0,.08)`。
- Hero 上半透明信息卡：`backdrop-filter: blur(8–12px)`（不支持则降级为实色半透明底）。

## 4. 布局与响应式

### 4.1 断点建议

| 断点 | 宽度 | 行为要点 |
| --- | --- | --- |
| 默认（移动） | &lt; 768px | 单列；总览卡横向滑动；时间线纵向堆叠 |
| 平板/桌面 | ≥ 768px | 总览可多列网格；时间线可双列或宽卡；hover 态生效 |

### 4.2 页面纵向信息架构（与 PRD 一致）

1. Hero  
2. 行程总览（信息卡）  
3. 出发前准备  
4. 每日时间线  
5. 路线可视化（与时间线联动）  
6. 美食推荐  
7. 避坑建议  
8. 总结与 CTA  

各模块顶部建议统一**小节标题 + 一句说明**，降低「不知道从哪看起」的痛点。

### 4.3 安全区与触控

- 主按钮最小点击区域约 **44×44px**（移动端）。
- 底部固定 CTA（若采用）需预留 `padding-bottom` 避免被系统手势条遮挡。

## 5. 全局组件规范

### 5.1 主按钮 / 次按钮

- **主按钮**：填充主色，白字；hover 略加深；focus 可见 `outline` 或 `box-shadow`。
- **次按钮**：描边或浅底；与主按钮并排时主按钮视觉更重。
- **行为**：「查看路线」→ 滚动锚点至时间线/路线区；「展开全部行程」→ 展开 Day1–Day6 详情（与 PRD 一致）。

### 5.2 标签（Chip）

- 小圆角胶囊；Hero 下标签示例：`自由行`、`路线清晰`、`可直接抄作业`（来自 PRD，可与页面清单主标题并存为副信息）。
- 关键词标签（夜景 / 老街 / 湖景…）：浅底 + 深色字，可横向换行或滑动区展示。

### 5.3 卡片

- 统一白底或浅表面色；标题 + 正文层级分明；可选左侧色条标识模块类型（仅作弱装饰）。

### 5.4 分段标题

- 模块级：`20–24px` 粗体 + 短副标题一行（灰色）。
- Day 内分段：`上午 / 下午 / 晚上 / 交通 / 美食 / tips` 使用小标题样式（全大写或字重区分二选一，保持全站一致）。

## 6. 分模块 UI 规格

### 6.1 Hero 首屏

| 元素 | UI 说明 |
| --- | --- |
| 背景 | 全宽大图（氛围图：象山夜景 / 日月潭 / 阿里山日出三选一）；`object-fit: cover`；可加自上而下轻微暗角或底部渐变，保证白字/浅字可读 |
| 半透明信息卡 | 叠于图下方或偏下 1/3；承载标题、副标题、标签、按钮；毛玻璃或 85–92% 不透明白底 |
| 主标题 | 页面清单：**台湾 6 天 5 晚互动攻略**（大字一行或两行断行） |
| 副标题 | **第一次去台湾，照着玩基本不踩雷** |
| 按钮组 | **查看路线**（主）+ **展开全部行程**（次）横向或纵向堆叠（移动纵向优先） |
| 动效 | 首屏整体 **150–400ms 淡入**（opacity + 轻微 translateY），避免过长影响 LCP 感知 |

### 6.2 总览信息卡

| 元素 | UI 说明 |
| --- | --- |
| 结构 | 2×2 或横向多卡：总天数、覆盖城市、推荐人群、关键词标签 |
| 桌面 | 网格排列；卡片 **hover** 边框色/阴影略增强 |
| 移动 | **横向滑动**（`scroll-snap` 可选），指示点或边缘渐变暗示可滑 |

### 6.3 出发前准备

| 元素 | UI 说明 |
| --- | --- |
| 标签栏 | 横向滚动或换行：**证件 / 交通 / 支付 / 必备工具**（PRD）；当前标签下划线或填充底 |
| 内容区 | 切换后展示对应卡片组；单卡可含**小图标 + 标题 + 短说明**（信息型图卡，见第 9 节） |
| 清单对齐 | 页面清单中的条目（电话卡、悠游卡、现金、Google Maps、入境单、车票提前预定）归入上述四类 |

### 6.4 行程时间线（Day1–Day6）

| 元素 | UI 说明 |
| --- | --- |
| 卡片列表 | 每日一卡：左侧竖条或圆点表示时间线节点；**选中/展开** 主色高亮 |
| 折叠态 | 显示 Day 序号、城市/主题一行、代表图缩略图（若有） |
| 展开态 | 展示 **上午 / 下午 / 晚上 / 交通 / 美食 / tips** 分块；块间留白充足 |
| 行程摘要（PRD） | Day1 台北：机场+西门町+象山；Day2 淡水；Day3 故宫+九份；Day4 日月潭；Day5 阿里山；Day6 日出+返程+桃园购物 —— 折叠态副标题可用 |

### 6.5 路线可视化区

| 元素 | UI 说明 |
| --- | --- |
| 形态 | 简化路径：**台北 → 淡水 → 九份 → 日月潭 → 阿里山 → 桃园**；折线、带节点的横条或「之」形折线（移动可用纵向蛇形） |
| 真实地图 | 节点条下方 **Google iframe** 概览 + 外链；节点「九份」地图 pin 为 **台北车站** |
| 预算计算器 | 总览「预算感参考」卡右下角 FAB，向上展开表单与结果区 |
| 节点 | 可点击圆点或城市 pill；**当前节点** 主色填充 + **轻微脉冲/发光**（CSS animation，节制） |
| 联动 | 点击节点 → 时间线滚动至对应 Day 并展开/高亮（与 PRD 一致） |

### 6.6 美食推荐区

| 元素 | UI 说明 |
| --- | --- |
| 卡片 | 店铺名醒目；可选食物缩略图区域 |
| 展开/翻转 | **二选一**：点击展开详情（实现成本低）或 3D flip（注意性能与无障碍）；背面/展开区：**推荐理由、场景、主观评分**（星级或 1–10） |
| 内容示例 | 品都串烧、天天利卤肉饭、葱爆山猪肉、民族烤肉/鸡翅包饭等（PRD） |
| 二刷 | 单独一行标签或图标：**是否值得二刷**（是/否/看情况） |

### 6.7 避坑建议区

| 元素 | UI 说明 |
| --- | --- |
| 分类 | **必做 / 可选 / 避坑** 三组或列表项左侧色标 + 文案 |
| 单条 | 短文 + 可选小图标；列表间距一致 |
| 示例文案 | 现金、公车票、跨城预留、回程班次、热门店预期管理等（PRD） |

### 6.8 总结 CTA 区

| 元素 | UI 说明 |
| --- | --- |
| 内容块 | **最推荐的 3 件事**（编号或图标列表）；**适合谁**（短段落或 bullet）；**真实体验声明**（小字免责声明风格） |
| 按钮 | **查看完整攻略**（外链小红书）；**复制路线**（配合剪贴板 API，失败时 toast 提示）；**访问 GitHub**（仓库链接） |
| 视觉 | 可与 Hero 主色呼应，形成「收尾回响」 |

## 7. 图片与媒体规范

### 7.1 类型（来自 PRD）

1. **氛围封面图**：Hero 唯一主视觉，情绪优先。  
2. **每日代表图**：每 Day 一张，建立「这一天什么感觉」。  
3. **信息型图卡**：准备事项类小图标或示意，提高扫描效率。

### 7.2 使用原则

- 每张图配**标题或说明**；`alt` 文本描述信息而非堆砌关键词。  
- 统一宽高比建议：Hero **16:9 或 4:5（移动）**；Day 卡 **4:3 或 1:1** 择一全站统一。  
- 资源放 `assets/`，相对路径引用。

## 8. 动效与微交互汇总

| 场景 | 建议参数 |
| --- | --- |
| 页面加载 | 首屏淡入 150–400ms |
| 卡片 hover（桌面） | 阴影/位移 ≤ 2px，过渡 150–200ms |
| Day 展开 | `height`/`grid` 动画或简单 opacity，200–300ms，避免布局抖动过大 |
| 路线当前节点 | 脉冲周期 2–3s，低幅度 |
| 标签切换 | 内容区 cross-fade 150ms 或即时替换（性能优先） |

可选：**深浅色切换** —— 通过 `data-theme` + CSS 变量切换背景与表面色，动效 200ms。

---

## 9. 无障碍与可读性

- 文本与背景对比度建议满足 **WCAG AA**（正文尤其）。  
- 可点击元素具备 `:focus-visible` 样式。  
- 路线节点、Day 选中不仅依赖颜色，辅以**字重、边框、图标**。  
- 动效尊重 `prefers-reduced-motion: reduce`：减弱或关闭淡入、脉冲。
